# 🙋‍♂️ เก็งข้อสอบ: คำถามที่อาจารย์มักจะถามตอนพรีเซนต์

จากโค้ดที่คุณเขียนในโปรเจกต์นี้ มีหลายจุดที่เป็น "หัวใจหลัก" ของ React ซึ่งอาจารย์มักจะเจาะถามเพื่อเช็คความเข้าใจครับ ผมรวบรวมคำถามยอดฮิตและ **"คำตอบแบบได้คะแนนชัวร์ๆ"** มาให้ครับ

---

## 1. ⚠️ คำถามเรื่อง: การจัดการ State และ Context (พบบ่อยที่สุด)

**❓ คำถาม:** ทำไมถึงเอาข้อมูล รายการโปรด (Favorites) ไปไว้ใน Context API แทนที่จะใช้ useState ธรรมดาส่งผ่าน props?
*   **✅ วิธีตอบ:** "เพราะข้อมูล Favorites มันต้องถูกใช้งานในหลายๆ หน้าครับอาจารย์ ทั้งใน Navbar (เพื่อโชว์ตัวเลข), ในหน้าแรก (ปุ่มหัวใจใน PostCard) และในหน้า FavoritesPage ด้วย ถ้าเราใช้ปกติต้องส่งข้อมูลย้อนไปย้อนมา (Prop Drilling) โค้ดจะวุ่นวายและดูแลยากมากครับ การใช้ Context เลยตอบโจทย์กว่าเพราะมันครอบแอปไว้เลย Component ไหนอยากใช้ก็ดึงไปได้ทันทีครับ"

**❓ คำถาม:** localStorage คืออะไร? ทำไมไม่เก็บข้อมูลไว้ใน State เฉยๆ
*   **✅ วิธีตอบ:** "State ปกติของ React เวลาเรารีเฟรชหน้าเว็บข้อมูลจะหายหมดเลยครับ (เพราะมันอยู่ใน RAM) ผมเลยใช้ localStorage (ความจำของบราวเซอร์) เพื่อให้ข้อมูลการกดถูกใจยังคงอยู่แม้ผู้ใช้จะปิดเว็บแล้วเปิดใหม่ครับ"

---

## 2. ⚠️ คำถามเรื่อง: การดึง API (Fetching & Hooks)

**❓ คำถาม:** ทำไมต้องเขียน Hook `useFetch` ขึ้นมาเอง? ทำไมไม่ Fetch ใน Component ไปเลย?
*   **✅ วิธีตอบ:** "เพื่อความ Reusable (การนำกลับมาใช้ซ้ำ) ครับ ในโปรเจกต์เรามีการดึงข้อมูลหลายจุด ทั้งดึงโพสต์ทั้งหมด และดึงโพสต์เดียว ถ้าเราเขียน Fetch ตรงๆ โค้ดจะซ้ำซ้อนและรกมากครับ ผมเลยแยก Logic การจัดการ Loading, Error และ Data ออกไปเป็น `useFetch` ทำให้ Component ปลายทางสะอาดและอ่านง่ายขึ้นมากครับ"

**❓ คำถาม:** ใน `useFetch` ตรง `useEffect` ทำไมต้องใส่ `[url]` ไว้ด้านหลัง (Dependency Array)?
*   **✅ วิธีตอบ:** "เพื่อบอก React ว่า 'ให้ดึงข้อมูลใหม่ทุกครั้งที่ค่า url เปลี่ยนแปลง' ครับ เช่น พอเรากดดูโพสต์ id 1 แล้วเปลี่ยนไปกดดู id 2 ค่า url เปลี่ยน useEffect ก็จะทำงานไปดึงข้อมูลใหม่มาให้ครับ ถ้าเราปล่อยว่าง `[]` มันจะดึงแค่ครั้งเดียวตอนโหลดเว็บครั้งแรกครับ"

---

## 3. ⚠️ คำถามเรื่อง: เทคนิคตอนดึงข้อมูลลึกๆ (Performance)

**❓ คำถาม:** ในหน้า FavoritesPage ทำไมถึงเลือกใช้ `Promise.all` ในการดึงข้อมูลโพสต์?
*   **✅ วิธีตอบ:** "เพราะเรามีรายการ ID ที่ผู้ใช้ถูกใจหลายอันครับ ถ้าเรา fetch ทีละอัน (แบบรัน loop ต่อคิวกัน) มันจะช้ามากเสียเวลาผู้ใช้ ผมเลยใช้ `Promise.all` เพื่อให้มันยิงคำขอ (Request) ไปดึงข้อมูลทุกโพสต์พร้อมๆ กันแบบคู่ขนาน (Parallel) ซึ่งมันจะทำให้หน้าเว็บโหลดเสร็จเร็วกว่ามากครับ"

**❓ คำถาม:** ทำไมความคิดเห็น (Comments) ถึงไม่โหลดมาพร้อมกับโพสต์ตั้งแต่แรก? (ในไฟล์ PostCard)
*   **✅ วิธีตอบ:** "เพื่อประหยัดเน็ตและลดภาระเซิร์ฟเวอร์ครับ (Lazy Fetching) ผมเขียน Logic ไว้ว่าให้โชว์แค่โพสต์ก่อน ถ้าผู้ใช้สนใจอยากอ่านคอมเมนต์และ 'กดปุ่มแสดงความคิดเห็น' โค้ดของผมถึงจะส่ง Request ไปดึงคอมเมนต์มาโชว์ครับ"

---

## 4. ⚠️ คำถามเรื่อง: Routing (React Router)

**❓ คำถาม:** React Router ทำให้เว็บเราต่างจากเว็บ HTML ปกติยังไง?
*   **✅ วิธีตอบ:** "มันทำให้เว็บเราเป็น SPA (Single Page Application) ครับ คือเวลาเราคลิกเปลี่ยนหน้าเว็บจากเมนู หน้าจอจะไม่กระพริบโหลดใหม่ (No Reload) เพราะ React จัดการสลับ Component ให้ไส้ในเปลี่ยนไปเอง ทำให้ผู้ใช้รู้สึกว่าเว็บไหลลื่นและเร็วกว่าเว็บยุคเก่าครับ"

**❓ คำถาม:** `useParams()` ในหน้า PostDetailPage เอาไว้ทำอะไร?
*   **✅ วิธีตอบ:** "เอาไว้ดูดค่าจาก URL ของบราวเซอร์ครับ เช่น ถ้า URL ปัจจุบันคือ `/posts/5` คำสั่ง useParams จะดึงเลข `5` ออกมาให้เราเอาไปใช้ดึง API ของโพสต์ที่ 5 ต่อไปได้ครับ"
